# FLAGOS LOG ANALYZER SKILL

This skill analyzes logs generated by the inference service in order to identify deployment or runtime issues.

Typical logs include:

- vLLM startup logs
- SGLang server logs
- CUDA runtime logs
- FlagGems related logs

The goal is to automatically identify common deployment problems and provide diagnostic feedback.

---

# WORKFLOW

Execute steps in order.

---

## STEP 1 — Locate Log Files

Identify possible log locations.

Common examples:

service.log  
vllm.log  
sglang.log  
nohup.out  

If the user started the service with nohup:

nohup <command> > service.log 2>&1 &

then logs are usually in:

service.log

If the log file location is unknown, ask the user to provide it.

Result feedback must include:

- detected log file path
- log size
- last modified time

---

## STEP 2 — Inspect Recent Log Output

Display the latest log lines:

tail -n 100 <log_file>

Focus on:

- startup sequence
- model loading
- GPU initialization
- server port binding

Result feedback:

- service startup status
- last log messages

---

## STEP 3 — Detect Common Startup Errors

Search for common failure keywords.

Example:

grep -i "error" <log_file>  
grep -i "cuda" <log_file>  
grep -i "oom" <log_file>  
grep -i "traceback" <log_file>

Typical failure types:

GPU memory issues  
CUDA driver mismatch  
missing model files  
tokenizer errors  
dependency conflicts

Result feedback:

- detected error category
- relevant log lines

---

## STEP 4 — Detect FlagGems Execution

Search for FlagGems execution messages.

Example:

grep -i "gems" <log_file>

Typical patterns:

flag_gems.ops  
GEMS MUL  
GEMS RECIPROCAL  

These logs indicate that FlagGems accelerated operators are being executed.

Result feedback:

- FlagGems detected or not
- related log entries

---

## STEP 5 — Detect GPU or Memory Errors

Search for GPU related problems.

Example keywords:

CUDA out of memory  
device not found  
driver mismatch  

Result feedback:

- GPU error status
- possible causes

---

## STEP 6 — Provide Diagnosis

Summarize findings.

Possible outcomes:

Service started successfully  
Service running but API unreachable  
Model loading failed  
GPU memory insufficient  
FlagGems not enabled  

Provide suggestions based on detected issues.

Examples:

reduce tensor parallel size  
check model path  
verify CUDA compatibility  
restart service with correct parameters

---

# COMPLETION CRITERIA

Log analysis is complete when:

- log file inspected
- errors categorized
- diagnosis generated
- possible solutions suggested
